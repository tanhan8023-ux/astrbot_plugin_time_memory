# 群聊时间与自动关键词记忆

这个 AstrBot 插件会在群聊里持续记录近期消息，但会匿名化保存说话者。它会自动调用当前 LLM 提取群内关键词，并把关键词直接加入当前群的关键词库。关键词命中后，插件不会自己抢着发第二条消息，而是把时间、群规则、关键词和群聊场景作为临时上下文注入给主聊天链路。

## 功能

- 感知北京时间，支持 `/time`、`现在几点`、`今天几号`。
- 自动收集近期消息和活跃度估计，说话者匿名化保存，不做群友画像。
- 每累计一定数量消息后，自动提取关键词、话题词、群内常聊梗。
- 关键词自动生效，也可以手动添加或删除；命中后只注入上下文，由主 LLM 或人设插件决定怎么回复。
- 支持“少说话”群规则，降低普通闲聊触发 LLM 的概率。

## 配置

在 AstrBot 插件配置里设置：

```yaml
timezone: Asia/Shanghai
trusted_user_ids: []
recent_message_limit: 80
keyword_extract_interval: 30
auto_keyword_limit_per_group: 50
quiet_reply_probability: 0.18
default_group_mode: normal
```

`trusted_user_ids` 在 AstrBot 面板里填写为字符串，多个 ID 用英文逗号分隔，例如 `123456,789012`。可以用 AstrBot 内置 `/sid` 查看自己的用户 ID。

本插件不会保存群友画像、好感度、用户备注或个人长期记忆。`trusted_user_ids` 只用于权限判断。

## 命令

- `/time`：查看当前北京时间。
- `/关键词`：查看当前群关键词库。
- `/添加关键词 <词>`：手动添加关键词。
- `/删除关键词 <编号或词>`：删除关键词，并短期阻止自动重新加入。
- `/群规则`：查看当前群规则。
- `/群记忆`：查看当前群摘要。
- `/time_memory_status`：查看插件状态。

白名单用户也可以自然地说：

- `添加关键词 原神`
- `删除关键词 原神`
- `记住这个群少说话`
- `让他少说话`
- `机器人少回复一点`
- `以后少回复一点`
- `恢复正常回复`

少说话规则只有 `trusted_user_ids` 里的白名单用户可以修改。若不是白名单用户触发，机器人会明确提示权限不足。

## 在 AstrBot 面板管理关键词

插件配置里有两个面板字段：

- `panel_keywords`：面板添加关键词。
- `panel_deleted_keywords`：面板删除/屏蔽关键词。

这两个字段在 AstrBot 面板里都填 JSON 字符串，格式示例：

最简单写法是一行一个词：

```text
原神
小手机
API
```

指定群号也可以这样写：

```text
123456789: 原神, 小手机
987654321: API
```

仍然兼容 JSON：

```json
{
  "*": ["全局关键词"],
  "123456789": ["只在这个群生效的关键词"]
}
```

`*` 表示所有群。群号键只影响对应群。保存配置并重载插件后生效。

## 数据文件

插件会在 `data/` 下保存：

- `group_memory.json`
- `group_keywords.json`
- `group_rules.json`

保存时会先写临时文件再替换正式文件，降低异常中断造成的数据损坏风险。

`group_memory.json` 中的新消息只保存匿名 `speaker`、时间和短文本；旧版本留下的 `user_id/user_name` 字段会被兼容读取，但不会在上下文或命令里展示。
